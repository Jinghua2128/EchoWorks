Data on this device

Guest progress is stored in this browser. It can include pulse answers, role and scenario history, scores, reflections, completion dates, and a random player identifier. It does not automatically move to another browser or device.

Firebase account and cloud data

Signed-in learners use Firebase Authentication and Firestore. Firestore stores the account email, selected role, completed scenarios, CARE or REAL scores, progress status, reflection answers, attempt details, and completion dates. Scenario scripts, survey questions, and AR card content remain local JSON files and are not stored in Firestore.

Dashboard access

Only explicitly authorised dashboard accounts can view learner analytics. Read-only viewers can inspect progress, scores, and reflections but cannot change learner records. Only the named dashboard owner can add or remove viewers, and the owner profile cannot be removed in the app.

Camera use

The browser asks for camera permission only after you choose Start camera on the AR page. Card recognition runs in the browser. Camera frames are not uploaded or saved by Feedback Playbook. The manual card viewer remains available without camera permission.

Retention and deletion

Learning records remain until you delete progress or the project owner removes them. Delete progress removes the learner's pulse answers, scenario progress, attempts, scores, and reflections from this device and, when signed in, from Firestore. It does not delete the Firebase Authentication account.

Email verification

New accounts receive a verification email. Learning can continue while verification is pending, but dashboard access always requires a separately authorised dashboard profile. Never share passwords or sign-in codes with project staff.
